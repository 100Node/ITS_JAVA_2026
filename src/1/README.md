## Контрольні питання

### 1. Призначення JFC і його архітектура

**JFC (Java Foundation Classes)** — це набір бібліотек для створення кросплатформних графічних інтерфейсів (GUI).

* **Архітектура:**  `AWT` (базовий набір, використовує вікна самої ОС).
* `Swing` (компоненти написані на Java, не залежать від ОС).
* `Java 2D` (малювання складної графіки).
* `Accessibility API` (доступність для людей з вадами).
* `Drag and Drop` (перетягування об'єктів).



---

### 2. Переваги Swing порівняно з AWT

* **Легковаговість (Lightweight):** компоненти малюються самою Java, а не операційною системою.
* **Кросплатформність:** програма виглядає ідентично на Windows, macOS та Linux.
* **Змінний дизайн (Pluggable Look and Feel):** можливість змінювати "теми" інтерфейсу на льоту.
* **Більше компонентів:** наявність складних елементів (таблиці `JTable`, дерева `JTree`, вкладки), яких немає в AWT.
* **Підтримка MVC:** розділення даних та їхнього візуального відображення.

---

### 3. Структура GUI програми на Java

Складається з таких рівнів:

1. **Головне вікно (`JFrame`):** основа програми.
2. **Проміжні контейнери (`JPanel`):** панелі для логічного групування елементів.
3. **Компоненти (`JButton`, `JLabel`):** самі кнопки, тексти, поля вводу.
4. **Менеджери компонування (Layouts):** правила, що вказують, як саме розмістити компоненти на панелі (в ряд, сіткою тощо).
5. **Обробники подій (Listeners):** код, який реагує на дії користувача (кліки).

---

### 4. Призначення Containers. Що є контейнерами?

**Призначення:** вміщувати інші графічні компоненти та керувати їхнім розташуванням за допомогою менеджерів компонування.
**Типи:**

* **Верхнього рівня** (самостійні вікна, не вкладаються нікуди): `JFrame`, `JDialog`, `JWindow`.
* **Проміжні** (вкладаються у вікна для групування компонентів): `JPanel`, `JScrollPane` (панель з прокруткою), `JTabbedPane` (вкладки).

---

### 5. Основні методи класу Frame/JFrame

* `setTitle("Текст")` — задає заголовок вікна.
* `setSize(ширина, висота)` — встановлює розміри.
* `setLayout(LayoutManager)` — задає правило розміщення елементів.
* `add(компонент)` — додає елемент або панель у вікно.
* `setVisible(true)` — показує вікно на екрані.
* `setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE)` — завершує програму при закритті вікна.

---

### 6. Призначення Panel і як додати Components

**Призначення `Panel` / `JPanel`:** це невидима прямокутна область для групування елементів. Складні інтерфейси будуються шляхом створення кількох панелей (наприклад, одна для меню, інша для кнопок), які потім додаються в головне вікно.
**Як додати:** за допомогою методу `add()`.
*Приклад:* `myPanel.add(myButton);`

---

### 7. Об’єкти event та механізм оброблення подій

* **Об'єкти event (події):** це контейнери (наприклад, `ActionEvent`, `MouseEvent`), що зберігають дані про дію користувача (яка кнопка натиснута, координати миші, час кліку).
* **Механізм (Модель делегування подій):**
1. **Джерело (Source)** — наприклад, кнопка, яку натиснули, генерує *Подію*.
2. **Подія (Event)** — передається системі.
3. **Слухач (Listener)** — фрагмент коду, який "чекав" на цю подію і тепер виконує відповідну дію (наприклад, зберігає файл).



---

### 8. Інтерфейси XxxListener: призначення та типи

**Призначення:** це контракти-шаблони. Щоб програма відреагувала на дію, ви повинні реалізувати відповідний інтерфейс і прописати логіку в його методах.
**Основні типи:**

* `ActionListener` — реакція на дію (натискання кнопки, Enter).
* `MouseListener` — кліки, наведення миші.
* `KeyListener` — натискання клавіш на клавіатурі.
* `WindowListener` — відкриття/закриття вікна.

**Приклад (реєстрація слухача для кнопки):**

```java
button.addActionListener(e -> System.out.println("Кнопку натиснуто!"));

```
